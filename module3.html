Nanopore based Data Analysis and AMR detection

  1. Prerequisite and Software Setup

1.1 Ubuntu: Installation (Windows)

  • Open Windows Powershell and run as administrator mode. Tip: search “powershell” in Windows search
  • To install WSL, run the command:

wsl --install
  • To set your root password, enter a new user and password
  • When installation is finished, reboot the machine.

1.2 Installing Custom Conda Environment (Miniconda)

  • Open Ubuntu terminal from Windows Start menu. Click on the “Windows” button and search for “ubuntu”. Clicking on the application will open an Ubuntu terminal, which is a LINUX terminal
  • If prompted, enter your username and password from step 1.
  • In this workshop, we will be using Miniconda (recommended as it is relatively small and quick to install) as a package manager to facilitate replication of working environments with all necessary software packages. To learn more about Conda, see: https://docs.conda.io/projects/conda/en/latest/user-guide/index.html  
  • Download the latest Miniconda installer for Linux:

wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
  • Run the installer:

bash Miniconda3-latest-Linux-x86_64.sh
  • Follow the prompts to complete the installation
  • Close and reopen the terminal to apply changes
  • If you have followed the instructions properly, you should see (base) at the beginning of the command line. This means that Miniconda is already activated.

  • Otherwise, activate conda:

conda activate

 

1.3 ONT Minknow and Guppy Installation

Depending on whether the user is basecalling on a CPU or a GPU, you will need to choose an appropriate Guppy package. These packages are available as separate installers on the Oxford Nanopore Technologies community website (https://community.nanoporetech.com).

  • CPU-only basecalling is available in the ont-guppy-cpu package.
  • GPU basecalling is available in the ont-guppy package and will require a GPU driver for basecalling to work. To install the GPU version of Guppy software for quicker basecalling, access the Software Downloads section on the Oxford Nanopore Technologies Community site (www.community.nanoporetech.com).
  • Login to Oxford Nanopore community website (https://community.nanoporetech.com) using nanopore login credential, login id and password (https://community.nanoporetech.com/).
  • Direct to “Software downloads” page from right selection options (https://community.nanoporetech.com/downloads).
  • Download Guppy (latest v6.5.7) for Linux 64-bit GPU download option.
  • Install Guppy to GPU enabled server by extracting the downloaded file named ont-guppy_6.5.7_linux64.tar, which is typically a tarball archive file, with the below command,

tar -xf ont-guppy_6.5.7_linux64.tar 

 

# Inside the .bash_profile or bashrc file, add the following line to append the bin folder containing Guppy executables as shown below to system's PATH variable.

nano ~/.bash_profile

export PATH="/path/to/ont-guppy/bin:$PATH"

# After saving the changes, apply them to your current terminal session by the following command to reload the configuration.

source ~/.bash_profile

 

 

  1. Basecalling and Demultiplexing

 

2.1 Basecalling using guppy_basecaller (High-Accuracy Model using GPU version)

To convert the raw signal data (FAST5) from the nanopore sequencer into readable nucleotide sequences (FASTQ) we will perform basecalling using Guppy Basecaller using the GPU version with the following commands. 

Note: In case of Oxford Nanopore GridION sequencing device, if basecalling has been performed with itself from device, then this step can be skipped and directly Demultiplexing can be done. 

# Without flowcell and kit name:
 
guppy_basecaller -c dna_r10.4.1_e8.2_400bps_hac.cfg -i /path/to/Runfolder/ -s ./basecalling/ -x auto 
# The auto can be replaced with CUDA cores for GPU-basecalling using,
guppy_basecaller -c dna_r10.4.1_e8.2_400bps_hac.cfg -i /path/to/Runfolder/ -s ./basecalling/ -x "cuda:1" -r
 
 
# With flowcell and kit name:
 
guppy_basecaller -c dna_r10.4.1_e8.2_400bps_hac.cfg -i /path/to/Runfolder/ -s ./basecalling/ --flowcell FLO-MIN114  
 --kit SQK-NBD114-24 -x auto -r 
# The auto can be replaced with CUDA cores for GPU-basecalling using,
guppy_basecaller -c dna_r10.4.1_e8.2_400bps_hac.cfg -i /path/to/Runfolder/ -s ./basecalling/ --flowcell FLO-MIN114 
 --kit SQK-NBD114-24 -x "cuda:1" -r
  • -c: Configuration file
  • -x: Specify GPU device.
  • -r: Search for input file recursively.
  • Substitute /path/to/Runfolder/ with folder where FAST5 files are present.

2.2 Demultiplexing using guppy_barcode (using GPU version)

As barcodes are used during library preparation, demultiplexing the reads into separate files for each barcode is required. We will use Guppy Barcoder here. This step will separate the reads belonging to different samples or barcodes. 

guppy_barcoder -i ./basecalling/ -s ./demultiplexing/ –barcode_kits "EXP-NBD114-24" -x auto
  • -i ./basecalling/: The input directory containing the basecalled reads (FASTQ files).
  • -s ./demultiplexing/: The output directory where demultiplexed reads will be saved.
  • --barcode_kits "EXP-NBD114-24": Specifies the barcode kit (in this case, EXP-NBD114).
  • -x auto: Uses GPU acceleration if available (-x auto will automatically select the best available hardware)
  1. Quality Control and Read Filtering

The raw reads need to be filtered fir low-quality reads, adapters, and barcodes.

3.1 Assessing Sequencing Statistics using NanoStat 

Finally, generate a summary of the sequencing run using NanoStat.

NanoStat --fastq filtered_reads.fastq.gz --outdir stats_output/
  • --fastq: Input filtered FASTQ file.
  • --outdir: Directory for saving the sequencing statistics

3.2 Adapter Trimming using Porechop

After basecalling, trim the adapter sequences from the FASTQ files using Porechop.

porechop -i basecalled_output/reads.fastq.gz -o trimmed_reads.fastq.gz --threads 4
  • -i: Input basecalled reads in FASTQ format.
  • -o: Output file for trimmed reads.
  • --threads: Number of CPU threads for faster processing.

3.3 Low-Quality Reads Filtering using NanoFilt

Use NanoFilt to filter out reads with a Phred score below 9 and sequences shorter than 200 bp.

zcat trimmed_reads.fastq.gz | NanoFilt -q 9 -l 200 | gzip > filtered_reads.fastq.gz
  • zcat: Decompresses the .fastq.gz file.
  • -q 9: Filters out reads with a Phred quality score below 9.
  • -l 200: Removes reads shorter than 200 bp.
  • The output is recompressed and saved as filtered_reads.fastq.gz.
  1. AMR Detection using ResFinder 

Provides a list of identified AMR genes, the level of coverage and identity, and statistics on the matches. It relies on the ResFinder database, which contains curated sequences of resistance genes for a wide range of antibiotic classes.

4.1 ResFinder Installation and Dependencies

# Clone the latest version and enter the resfinder directory

git clone https://github.com/cadms/resfinder.git

cd resfinder && make

 

Depending on how you plan to run ResFinder, either BLAST or KMA options can be used. BLAST is used to analyse assemblies (i.e. FASTA files). KMA is used to analyse read data (i.e. FASTQ files).

4.1.1 Python modules: Tabulate, BioPython, CGECore and Python-Git

To install the needed python modules you can use pip.

pip3 install tabulate biopython cgecore gitpython python-dateutil

 

4.2 KMA

KMA is a mapping method designed to map raw reads directly against redundant databases. The instructions here will install KMA in the default location ResFinder uses. KMA can be installed in another location but the path to KMA will then need to be specified every time you run ResFinder unless you add the kma program to your PATH.

# Go to the directoy in which you installed the ResFinder tool

cd /path/to/some/dir/resfinder

cd cge

git clone https://bitbucket.org/genomicepidemiology/kma.git

cd kma && make


4.3 Databases

This section describes how to install the databases at the ResFinder default locations. The database locations can be changed, but must then be specified to ResFinder at run time.

4.3.1 ResFinder database

Provides matches of resistance genes with details like identity, coverage, and alignment scores.

# Go to the directory in which you installed the ResFinder tool

cd /path/to/dir/resfinder

git clone https://git@bitbucket.org/genomicepidemiology/resfinder_db.git db_resfinder

# Indexing databases

cd db_resfinder

python3 INSTALL.py

  

4.3.2 PointFinder database

Detects specific point mutations within genes that are known to cause antimicrobial resistance.

# Go to the directory in which you installed the ResFinder tool

cd /path/to/dir/resfinder

git clone https://git@bitbucket.org/genomicepidemiology/pointfinder_db.git db_pointfinder

# Indexing databases

cd db_pointfinder

python3 INSTALL.py

 

 

4.4 AMR Detection

Command you are using with KMA (k-mer alignment tool) for antimicrobial resistance (AMR) detection in Nanopore reads.

kma -i nanoporeReads.fq.gz -o amr_out -t_db ./resfinder/db_resfinder/all -mem_mode -mp 20 -mrs 0.0 -bcNano -bc 0.7

 

·  -i nanoporeReads.fq.gz: Input file containing the raw Nanopore reads in FASTQ format.

·  -o amr_out: Output file prefix (results will be saved as amr_out.*).

·  -t_db ./resfinder/db_resfinder/all: Specifies the ResFinder AMR database.

·  -mem_mode: Enables memory mode, suitable for large datasets.

·  -mp 20: Sets the minimum posterior value to 20 for the mapping.

·  -mrs 0.0: Sets minimum relative score to 0.0, allowing all reads to be considered.

·  -bcNano: Specifically optimized for Nanopore reads, taking into account their error profile.

·  -bc 0.7: Sets the minimum coverage to 70% for reporting a hit.
