AMR detection in Illumina based sequencing reads

 

  1. Background

Based on sequencing platforms, a variety of computational methods are used to detect AMR determinants, and understand evolutionary processes like mutations and horizontal gene transfer (HGT), which are key drivers of resistance. Some of the key methods include:

  • Variant calling and analysis: This involves identifying single nucleotide polymorphisms (SNPs), insertions, deletions, and other genetic variations that contribute to AMR.
  • Gene annotation and resistance gene identification: Tools like CARD (Comprehensive Antibiotic Resistance Database) or ResFinder are used to map resistance genes within bacterial genomes.
  • Phylogenetic analysis: By constructing phylogenetic trees, researchers can track the evolution and spread of AMR strains across different regions or host populations.
  • Horizontal gene transfer (HGT) detection: Specialized tools are used to identify genes that have been acquired through HGT, a major mechanism by which bacteria acquire AMR traits from other species.

Through the integration of these sequencing technologies and bioinformatics methods, researchers can not only detect and monitor AMR but also gain a deeper understanding of how resistance evolves, spreads, and can be mitigated.

1.1 Variant Calling

Variant calling is the process of identifying genetic variants, such as single nucleotide polymorphisms (SNPs), insertions, deletions (INDELs), and structural variants (SVs), from sequencing data. This is crucial for detecting mutations that confer resistance to antibiotics.

  • Steps in Variant Calling:
  1. Alignment: Raw sequence reads are aligned to a reference genome using tools like BWA, Bowtie2, or Minimap2.
  2. Calling Variants: Variants are called using tools like GATK (Genome Analysis Toolkit), FreeBayes, or Samtools mpileup.
  3. Filtering and Annotation: Variants are filtered based on quality metrics and then annotated using tools like SnpEff or VEP (Variant Effect Predictor) to determine their functional effects, including any impact on drug resistance genes.

1.2 Antimicrobial Resistance Gene Detection

Detection of AMR genes in bacterial genomes is done by comparing the genome sequence to known resistance gene databases. This process helps to identify genes responsible for enzymatic degradation, efflux pumps, and other resistance mechanisms.

  • Tools and Databases:
  1. CARD (Comprehensive Antibiotic Resistance Database): A curated database of resistance genes.
  2. ResFinder: A tool that identifies acquired antimicrobial resistance genes using BLAST against a curated database.
  3. ARIBA: A rapid tool for identifying AMR genes and mutations from sequencing data.
  4. AMRFinderPlus: A tool for detecting AMR genes using BLAST or HMM (Hidden Markov Model)-based searches against a large database of resistance genes.

1.3 Whole Genome Assembly and Annotation

Genome assembly involves constructing a consensus genome sequence from raw sequencing reads, with or without using a reference genome. When no reference genome is used, it’s called de novo genome assembly. Once assembled, the genome can be annotated to identify genes, including AMR genes.

  • Tools for de novo genome Assembly:
    • SPAdes: Commonly used for bacterial genome assembly.
    • Velvet: Another assembler often used for small genomes like bacteria.
    • Unicycler: Specialized for bacterial genome assembly, particularly for hybrid assembly (short and long reads).
  • Genome Annotation:
    • Prokka: Rapid annotation of prokaryotic genomes, including identification of AMR genes.
    • RAST (Rapid Annotation using Subsystem Technology): Another tool for genome annotation, which provides functional categories for genes.

1.4 Phylogenetic Analysis

Phylogenetic analysis helps to infer evolutionary relationships between bacterial strains, allowing researchers to track the spread of resistance genes across populations or geographical regions.

  • Tools for Phylogenetics:
    • IQ-TREE: A fast and efficient tool for constructing phylogenetic trees from sequence data.
    • RAxML: Commonly used for large-scale phylogenetic tree construction.
    • MEGA: A user-friendly tool for evolutionary analysis.

1.5 KEGG Pathway Enrichment Analysis

KEGG Enrichment Analysis is a statistical method used to identify KEGG pathways that are significantly overrepresented in a given set of genes or proteins. This analysis helps in understanding the biological significance of gene expression data, identifying pathways that are potentially involved in specific biological processes or diseases.

  • Tools for KEGG ortholog annotation
    • KAAS: KEGG Automatic Annotation Server provides functional annotation of genes by BLAST or GHOST comparisons against the manually curated KEGG GENES database.
    • eggNOG-mapper:  functional annotation tool for large sets of sequences based on fast orthology assignments.
    • BlastKOALA:KEGG's internal annotation tool for KO (K number) assignment using SSEARCH computation.
       
       
  • Tools for KEGG pathway enrichment analysis
    • getENRICH: Designed to facilitate comprehensive gene enrichment analysis, particularly for non-model organisms.
    • DAVID: provides a comprehensive set of functional annotation tools for investigators to understand the biological meaning behind large lists of genes. 
    • ShinyGO: ShinyGO based on a large annotation database derived from Ensembl and STRING and KEGG.

1.6 Comparative Genomics

Comparative genomics involves comparing the genomes of different bacterial strains or species to identify genetic differences, including AMR determinants, mobile genetic elements (e.g., plasmids), and virulence factors.

  • Tools for Comparative Genomics:
    • Roary: A tool to perform pangenome analysis, identifying core and accessory genes across multiple genomes.
    • Mauve: Used for genome alignment and identifying large-scale genomic rearrangements.
  1. Illumina based AMR detection analysis

We’ll be working on an example dataset of Acinetobacter baumannii, retrieved from NCBI SRA, where raw genome data is publicly available and will be performing variant calling and genome assembly and annotation. We will be making use of Conda package manager to download and install tools and perform our analysis:

We’ll be retrieving dataset SRR30953553 from NCBI SRA (https://www.ncbi.nlm.nih.gov/sra) and downloading it using the following code:

# We’ll be creating a new conda environment and activating it using the following code

 conda create -n amr 
 conda activate amr

 

# We’ll be installing the sra-tools utility to download our dataset

 conda install -c bioconda sra-tools

 

# Now we’ll download and extract our data using the following command

# We should get 2 output files in. fastq format as output

prefetch SRR30953553

fastq-dump –split-files SRR30953553/SRR30953553.sra

 

# We’ll compress this FASTQ data to speed up our analysis and save space using:

gzip SRR30953553_1.fastq SRR30953553_2.fastq

 

# Once the data is downloaded, we’re ready to perform quality control 

#(QC) step, first, we’ll assess the quality of our data using FASTQC

 

# FASTQC

 conda install -c bioconda fastqc 

 

# Create an output directory to store your results in: 

mkdir fastqc

  

# Run FASTQC using: 

fastqc -t 4 -o fastqc SRR30953553_1.fastq.gz SRR30953553_2.fastq.gz

 

# The FASTQC report contains comprehensive information, and a manual can #be found at: #https://www.bioinformatics.babraham.ac.uk/projects/fastqc/Help/

# Now, we’ll perform the next step of our QC, removing adapter sequences #and low-quality reads using a tool called Trim-galore

# Install

conda install -c bioconda trim-galore

 

# Now we’ll trim our data 

trim_galore -o trimmed_data -j 4 —length 30 –paired –fastqc SRR30953553_1.fastq.gz SRR30953553_2.fastq.gz

 

# This will result in new FASTQ files (.fq.gz) being created, including #a FASTQC report of your data after trimming. 

# Now our data is filtered, we’ll use it to perform the next step, # Called mapping/alignment, where we’ll align our data against a #reference genome. First, we’ll install the tools needed for downloading # Our reference and mapping, then we’ll perform the step.

conda install -c bioconda ncbi-datasets-cli bwa samtools

 

# Now we’ll download the A. baumannii ATCC 19606 reference genome from #NCBI using the following command: 

datasets download genome accession GCF_009035845.1 

 

# We’ll get a ncbi_dataset.zip file, which we’ll decompress and copy our # reference genome in FASTA (.fna) format to the present directory. 

unzip ncbi_dataset.zip 

cp ncbi_dataset/data/GCF_009035845.1/GCF_009035845.1_ASM903584v1_genomic.fna .

 

# Firstly, we’ll index our reference genome

bwa index GCF_009035845.1_ASM903584v1_genomic.fna

 

# Now, we’ll perform alignment

 samtools view -bS mapped.sam > mapped.bam 

 

# This will result in a .SAM (Sequence Alignment Map) format file containing alignment sequences and their quality. For faster processing and decreasing space, we’ll convert the SAM format to BAM (Binary Alignment Map) using Samtools. 

bwa mem -t 4 GCF_009035845.1_ASM903584v1_genomic.fna trimmed_data/SRR30953553_1_val_1.fq.gz trimmed_data/SRR30953553_2_val_2.fq.gz > mapped.sam

 

# We’ll assess our mapping stats using: 

samtools flagstat mapped.bam

 

# Now that our data is aligned, we can perform Variant Calling and identify SNPs and INDELs. We’ll be doing that using Freebayes. 

conda install -c bioconda freebayes

 

# We need a BAM file sorted by reference position in order to call #variants by Freebayes, we can do that by the following command: 

samtools sort -o mapped.sorted.bam mapped.bam

 

# Now that we have a sorted BAM file, we’ll call variant by providing #our reference FASTA file and our sorted BAM file: 

Freebayes -f GCF_009035845.1_ASM903584v1_genomic.fna mapped.sorted.bam > variants.vcf

 

# The next step in order to find AMR genes is to perform genome assembly 

# We’ll be using Spades to perform de novo genome assembly

conda install -c bioconda spades

 

# We’ll be using our filtered data as input to perform genome assembly. 

spades.py -t 4 -o assembly -1 trimmed_data/SRR30953553_1_val_1.fq.gz -2 trimmed_data/SRR30953553_2_val_2.fq.gz 

 

# The contigs.fasta file, in our assembly directory, contains the genome assembly. We’ll search for AMR genes in this dataset using AMRFinderPlus, since it can take both nucleotide and protein FASTA files as input

conda install -c bioconda ncbi-amrfinderplus 

 

# Now, we’ll set up our database using

amrfinder -u

 

# Now, we’ll run amrfinder

amrfinder --nucleotide assembly/contigs.fasta --organism Acinetobacter_baumannii --threads 4 --output amr_genes.txt 

 

# In order to retrieve amino acid sequences, we need to annotate our #genome assembly. We’ll achieve this by using PROKKA.

 

# Install PROKKA

conda install -c bioconda prokka

 

# We’ll run PROKKA using:

prokka --outdir a_baumannii --force --prefix A_baumannii --compliant --cpus 4 --norrna --notrna assembly/contigs.fasta 

# The PROKKA output will contain various types of files, including nucleotide and amino acid FASTA.
